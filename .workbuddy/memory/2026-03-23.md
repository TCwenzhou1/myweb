## Next.js 项目构建错误修复

### 问题背景
用户的 Next.js 项目存在 2 个构建错误，需要修复但不改变整体设计。

### 错误详情和修复

#### 1. AboutMe.tsx 第 79 行 - react/no-unescaped-entities
**原问题**:
- JSX 文本中有未转义的英文双引号 `"`
- 导致 ESLint 规则 `react/no-unescaped-entities` 失败

**原代码**:
```jsx
<span className="text-accent font-medium">"把想法做成系统"</span>
```

**修复方案**:
- 使用中文括号「」替换英文双引号
- 保持文本内容和视觉效果不变

**修复代码**:
```jsx
<span className="text-accent font-medium">「把想法做成系统」</span>
```

**优势**:
- ✅ 符合 ESLint 规则
- ✅ 中文文案排版更专业
- ✅ 无需转义处理
- ✅ 视觉效果更好

#### 2. Hero.tsx 第 107 行 - react/jsx-no-comment-textnodes
**原问题**:
- JSX 标签内的 `//` 被识别为注释而非文本
- 导致 ESLint 规则 `react/jsx-no-comment-textnodes` 失败

**原代码**:
```jsx
<div className="animate-pulse">// 持续构建中...</div>
```

**修复方案**:
- 将文本用 `{}` 花括号包装
- 作为 JavaScript 表达式正确输出

**修复代码**:
```jsx
<div className="animate-pulse">{'// 持续构建中...'}</div>
```

**优势**:
- ✅ 显式声明为 JavaScript 表达式
- ✅ 符合 React 最佳实践
- ✅ 提高代码可读性
- ✅ 符合 ESLint 规则

## Vercel 构建依赖修复（第二部分）

### 问题背景
Vercel 自动部署时报错，无法找到 `clsx` 和 `tailwind-merge` 模块。

### 根本原因
`package.json` 缺少以下生产依赖：
- `clsx@^2.0.0` - CSS 类名合并工具
- `tailwind-merge@^2.2.0` - Tailwind CSS 冲突解决方案

这两个包在 `lib/utils.ts` 中被导入使用，但未在 `package.json` 的 `dependencies` 中声明。

### 修复方案

**文件修改**: `package.json` (第 11-18 行)

**修改前**:
```json
  "dependencies": {
    "next": "14.2.5",
    "react": "^18",
    "react-dom": "^18",
    "lucide-react": "^0.344.0"
  },
```

**修改后**:
```json
  "dependencies": {
    "next": "14.2.5",
    "react": "^18",
    "react-dom": "^18",
    "lucide-react": "^0.344.0",
    "clsx": "^2.0.0",
    "tailwind-merge": "^2.2.0"
  },
```
